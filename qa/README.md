# QA 持续优化流程

## 流程

```
玩家角色体验 → 记录问题 → 分析优先级 → 实施改进 → 回归验证 → 下一轮
```

## 目录结构

```
qa/
├── personas/          # 玩家角色定义
│   ├── casual.md      # 休闲玩家
│   ├── hardcore.md    # 硬核玩家
│   ├── newbie.md      # 新手玩家
│   ├── mobile.md      # 手机端玩家
│   └── strategist.md  # 策略型玩家
├── rounds/            # 每轮测试记录
```

## 玩家角色

| 角色 | 关注点 | 测试方式 |
|------|--------|----------|
| 休闲玩家 | 上手难度、趣味性、单局时长 | 不看教程直接玩 |
| 硬核玩家 | 数值平衡、技能深度、操作上限 | 反复对比不同Build |
| 新手玩家 | 引导清晰度、UI易读性 | 第一次接触游戏 |
| 手机玩家 | 触控体验、按钮大小、性能 | 模拟手机触屏操作 |
| 策略玩家 | 职业差异、属性意义、进阶路线 | 分析数值和机制 |

## 评分标准

每项 1-5 分：
- **上手体验**: 新玩家能否在30秒内理解核心玩法
- **操控手感**: 移动和技能释放是否流畅
- **数值反馈**: 伤害/升级/技能效果是否有明显感知
- **视觉清晰**: 能否快速区分敌我/安全区/技能效果
- **策略深度**: 不同选择是否带来不同体验
